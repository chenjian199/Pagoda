# Pagoda Mocker 迁移裁剪与继承清单

## 1. 文档目的

本文档给出从当前 Dynamo `lib/mocker` 到 Pagoda `mocker` 的迁移边界。

目标：

```text
继承当前已经实现且仍适用的模块；
删除与 Pagoda 当前范围不符的模块；
替换 KVBM/memory 相关实现；
新增 LMCache-only 模拟能力；
保证文档与目标代码一致。
```

## 11. 行为兼容要求

虽然模块被裁剪，以下行为不能改变：

```text
相同 tokens + prefix caching on:
  应产生相同 full block identity；
  后续请求可命中 prefix。

prefix caching off:
  相同 tokens 不应跨请求复用。

chunked prefill:
  token budget 不足时可分 pass 继续。

preemption:
  容量不足时释放 victim active blocks；
  victim 重新等待。

decode:
  每个 ready request 每 pass 生成一个 token；
  到 max_output_tokens 后完成。

events:
  新 block 才发 Stored；
  active/inactive hit 不重复发 Stored；
  真正淘汰才发 Removed。

replay:
  offline 逻辑时钟可重复；
  online 使用 wall-clock。
```

## 12. 风险点

### 12.1 去掉 kvbm-logical 后 eviction 语义变化

原模块使用 lineage-aware backend 作为默认 eviction。  
Pagoda 如果改成简单 LRU，需要在文档和测试中明确。

### 12.2 LMCache mock 与真实 LMCache 不一致

Mock 只验证控制面与路由收益，不验证真实字节传输。

### 12.3 ExternalShared event 语义

如果 router 把 ExternalShared 当成本地 Device 命中，会造成过度乐观。  
必须在 kv_router cost model 中区分层级权重。

### 12.4 Replay 与 live 事件可见性

Offline 同步 apply events，Live deferred publish。  
两者语义要通过测试对齐。
