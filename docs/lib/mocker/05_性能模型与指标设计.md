# Pagoda Mocker 性能模型与指标设计

## 1. 模块目标

本文件定义 Pagoda Mocker 中 prefill/decode 时间预测、模拟速度控制、forward pass metrics 和 replay report 指标设计。

该部分继承原 `common/perf_model.rs`、`scheduler/mod.rs`、`replay/collector.rs` 的核心能力。

---

## 2. PerfModel

Pagoda 保留三种性能模型：

```rust
enum PerfModel {
    Polynomial,
    Interpolated { ... },
    Aiconfigurator { ... },
}
```

### 2.1 Polynomial

默认模型。

Prefill：

```text
tokens = batch_size * new_tokens_per_request
time_ms = a * tokens^2 + b * tokens + c
```

作用：

```text
不依赖 profile 文件；
适合 CI；
适合快速测试；
与真实硬件只保持趋势近似。
```

Decode：

```text
active_perc = active_kv_tokens / total_kv_tokens
time_ms = polynomial(active_perc)
```

并且至少返回 1ms，避免 token emission 落在同一时间戳。

### 2.2 Interpolated

从 NPZ profile 文件加载插值表。

需要数组：

```text
prefill_isl
prefill_ttft_ms
decode_active_kv_tokens
decode_context_length
decode_itl
```

Prefill 使用 1D interpolation。  
Decode 使用 2D interpolation。

用途：

```text
用真实 profile 数据提升模拟精度。
```

### 2.3 Aiconfigurator

通过 callback 调用外部 AIC SDK。

Prefill 输入：

```text
batch_size
new_tokens_per_request
prefix_tokens
```

Decode 输入：

```text
batch_size
context_length
output_length = 2
```

Pagoda 第一阶段只要求 vLLM backend 可用。

## 3. Prefill 时间计算

调度器对每个 pass 聚合：

```text
batch_count
batch_total_isl
batch_total_prefix
```

计算：

```text
mean_isl = batch_total_isl / batch_count
mean_prefix = batch_total_prefix / batch_count
new_tokens = mean_isl - mean_prefix
```

然后：

```text
prefill_ms = perf_model.predict_prefill_time(batch_count, mean_isl, mean_prefix)
```

再应用：

```text
prefill_ms /= speedup_ratio
```

特殊情况：

```text
batch_count == 0:
  prefill_ms = 0

worker_type == Decode:
  prefill_ms = 0
```

---

## 4. Decode 时间计算

Decode 输入：

```text
batch_size = ready requests 数量
active_kv_tokens = active_blocks * block_size
total_kv_tokens = num_gpu_blocks * block_size
context_length = ready requests 平均序列长度
```

然后：

```text
decode_ms = perf_model.predict_decode_time(
    batch_size,
    active_kv_tokens,
    context_length,
    total_kv_tokens
)
```

再应用：

```text
decode_ms /= speedup_ratio * decode_speedup_ratio
```

特殊情况：

```text
worker_type == Prefill:
  decode_ms = 0
```

因为 prefill worker 的首 token 输出视为 prefill pass 的一部分。

## 5. LMCache 传输与命中延迟

Pagoda 新增 LMCache 模拟后，需要在性能模型之外叠加外部缓存延迟。

### 5.1 LMCache hit latency

```text
L1 hit:
  增加 l1_hit_latency_ms

L2 hit:
  增加 l2_hit_latency_ms
```

应用点：

```text
LMCache prefix hit 发生在 prefill 前；
如果命中需要加载到本地 Device cache，应计入本 pass end_ms。
```

### 5.2 LMCache save latency

```text
新 full block 保存到 LMCache 时增加 save_latency_ms。
```

第一阶段建议：

```text
save_latency 不阻塞当前 decode 输出；
但影响 ExternalShared Stored event 可见时间。
```

如果实现简化：

```text
save_latency_ms = 0 时立即可见；
save_latency_ms > 0 时 pass end 可见。
```

### 5.3 P/D handoff delay

继承原函数：

```text
delay_ms = input_tokens * kv_bytes_per_token / bandwidth
```

用于 prefill worker 输出 `handoff_delay_ms`。

## 6. speedup_ratio

保留两个速度控制参数：

```rust
speedup_ratio
decode_speedup_ratio
```

语义：

```text
speedup_ratio:
  同时加速 prefill 和 decode。

decode_speedup_ratio:
  只加速 decode。
```

有效 decode 加速：

```text
effective_decode_ratio = speedup_ratio * decode_speedup_ratio
```

如果 ratio <= 0，则不缩放。

---

## 7. ForwardPassSnapshot

每个 pass 结束后生成：

```rust
ForwardPassSnapshot {
    num_prefill_requests,
    sum_prefill_tokens,
    var_prefill_length,
    sum_prefill_kv_tokens,
    num_decode_requests,
    sum_decode_kv_tokens,
    var_decode_kv_tokens,
    num_queued_prefill,
    sum_queued_prefill_tokens,
    var_queued_prefill_length,
    num_queued_decode,
    sum_queued_decode_kv_tokens,
    var_queued_decode_kv_tokens,
    wall_time_secs,
}
```

### 7.1 scheduled_prefills

对本 pass 中执行了 prompt prefill 的请求统计：

```text
prompt_len
prefix_tokens
tokens_computed
```

### 7.2 scheduled_decodes

对本 pass 中执行 decode 的请求统计：

```text
sequence_len
```

### 7.3 queued_prefills

等待队列中仍处于 Waiting 的请求。

### 7.4 queued_decodes

等待队列中 Preempted 的请求，视为 queued decode。

## 8. Welford 方差

保留 Welford 在线算法，用于计算 population variance：

```text
prefill length variance
decode KV token variance
queued prefill variance
queued decode variance
```

设计原因：

```text
避免保存所有样本；
与 Python 侧 ForwardPassMetrics 语义对齐。
```

---

## 9. MockerMetrics

Live scheduler 对外提供：

```rust
struct MockerMetrics {
    dp_rank,
    active_decode_blocks,
    total_blocks,
    gpu_cache_usage_perc,
}
```

更新时机：

```text
每个 pass 完成后更新 watch channel。
```

用途：

```text
router active load；
planner 观察；
测试断言；
监控展示。
```

---

## 10. TraceCollector 与 Report

Replay 过程中收集：

```text
arrival
admission
token emission
completion
reused_input_tokens
```

最终生成 `TraceSimulationReport`。

报告内容包括：

```text
request counts
throughput
TTFT
TPOT / ITL
end-to-end latency
distribution stats
percentiles
stddev
reuse stats
```

Pagoda 第一阶段应保留该报告能力。

---

## 11. 测试要求

```text
1. Polynomial prefill 随 token 增大；
2. Polynomial decode 至少 1ms；
3. speedup_ratio 缩短 prefill/decode；
4. decode_speedup_ratio 只影响 decode；
5. Interpolated NPZ 维度校验；
6. AIC callback 被调用；
7. ForwardPassSnapshot scheduled prefill 统计正确；
8. queued prefill / queued decode 统计正确；
9. MockerMetrics active_decode_blocks 更新；
10. LMCache L1/L2 hit latency 计入 pass；
11. handoff_delay_ms 按 kv_bytes_per_token 和 bandwidth 计算；
12. TraceCollector 生成 report。
```
